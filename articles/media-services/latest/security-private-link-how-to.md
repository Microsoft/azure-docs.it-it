# <a name="create-a-media-services-and-storage-account-with-a-private-link"></a>Creare un account di Servizi multimediali e archiviazione con un collegamento privato

[!INCLUDE [media services api v3 logo](./includes/v3-hr.md)]

Creare un account di Servizi multimediali e un account di archiviazione con collegamenti privati a una rete virtuale. Il modello Azure Resource Manager (ARM) configura anche DNS per entrambi i collegamenti privati. Infine, il modello crea una macchina virtuale per consentire all'utente di provare i collegamenti privati.

## <a name="prerequisites"></a>Prerequisiti

Leggere [Guida introduttiva: Creare e distribuire modelli di Arm usando il portale di Azure](https://docs.microsoft.com/azure/azure-resource-manager/templates/quickstart-create-templates-use-the-portal).

## <a name="limitations"></a>Limitazioni

- Per Servizi multimediali, il modello configura solo Collegamento privato per il recapito delle chiavi.
- Non viene creato un gruppo di sicurezza di rete per la macchina virtuale.
- Il controllo di accesso alla rete non Ã¨ configurato per l'account di archiviazione o il recapito delle chiavi.

Il modello crea:

- Un account di Servizi multimediali e un account di archiviazione (come di consueto)
- Una rete virtuale con una subnet
- Per l'account di Servizi multimediali e l'account di archiviazione:
  - Endpoint privati
  - DNS privato zone
  - Collegamenti tra collegamenti (per connettere le zone DNS private alla rete virtuale)
  - DNS privato di zona (per attivare la creazione automatica di record DNS nelle zone DNS private)
- Una macchina virtuale (con l'indirizzo IP pubblico e l'interfaccia di rete associati)
