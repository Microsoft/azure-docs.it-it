# <a name="create-and-manage-records-in-dynamics-365-by-using-azure-logic-apps"></a>Creare e gestire record in Dynamics 365 usando app per la logica di Azure

Dynamics 365 usa il [Common Data Service](https://docs.microsoft.com/powerapps/maker/common-data-service/data-platform-intro). Per le connessioni a Dynamics 365, usare il [connettore Common Data Service](https://docs.microsoft.com/connectors/commondataservice/).

> [!IMPORTANT]
> Il [connettore Dynamics 365](https://docs.microsoft.com/connectors/dynamicscrmonline/) è deprecato, ma continua a funzionare fino a quando non viene rimosso. Non usare il connettore Dynamics 365 per le nuove app per la logica. Per la rimozione del connettore Dynamics 365 non è ancora stata annunciata alcuna sequenza temporale. Non è necessario convertire le app per la logica esistenti nel connettore di Common Data Service o in un altro nuovo connettore pianificato, ma è necessario convertire le app per la logica quando il connettore viene rimosso. Per ulteriori informazioni, vedere il [connettore Dynamics 365 è deprecato](https://docs.microsoft.com/power-platform/important-changes-coming).
>
> Il [connettore di Common Data Service](https://docs.microsoft.com/connectors/commondataservice/) offre le stesse funzionalità del connettore di Dynamics 365 deprecato, ma include miglioramenti che aumentano l'affidabilità. Per informazioni sull'uso del connettore di Common Data Service nelle app per la logica, vedere [Creare e gestire record di Common Data Service usando App per la logica di Azure](../connectors/connect-common-data-service.md).

Per ulteriori informazioni su Common Data Service, vedere gli argomenti seguenti:

* [Informazioni: Introduzione a Common Data Service](https://docs.microsoft.com/learn/modules/get-started-with-powerapps-common-data-service/)
* [Informazioni: connettere e analizzare i dati di Dynamics 365 usando la piattaforma Power e Common Data Service](https://docs.microsoft.com/learn/wwl/connect-analyze-dynamics-365-data/)
