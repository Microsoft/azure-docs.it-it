# <a name="scenario-apache-tez-application-hangs-in-azure-hdinsight"></a>Scenario: Apache Tez application hangs in Azure HDInsight

Questo articolo descrive i passaggi per la risoluzione dei problemi e le possibili soluzioni per i problemi relativi all'interazione con i cluster HDInsight di Azure.This article describes troubleshooting steps and possible resolutions for issues when interacting with Azure HDInsight clusters.

## <a name="issue"></a>Problema

Dopo aver inviato il processo Apache Hive, dalla visualizzazione Tez lo stato del processo è "In esecuzione", ma non sembra procedere

## <a name="cause"></a>Causa

Troppi posti di lavoro presentati; long Yarn queue.

## <a name="resolution"></a>Risoluzione

Aumentare la scalabilità verticale del cluster o semplicemente attendere che la coda di Filati venga scaricata.

Per `yarn.scheduler.capacity.maximum-applications` impostazione predefinita controlla il numero massimo di applicazioni `10000`in esecuzione o in sospeso e il valore predefinito è .
