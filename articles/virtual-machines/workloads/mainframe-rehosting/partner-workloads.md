# <a name="mainframe-workloads-supported-on-azure"></a>Carichi di lavoro mainframe supportati in Azure

Le soluzioni elencate di seguito consentono di eseguire la migrazione del carico di lavoro mainframe Microsoft Azure. La migrazione di alcuni carichi di lavoro può essere eseguita con relativa facilità. È possibile eseguire il rehosting di altri carichi di lavoro che dipendono da software di sistema legacy. 

Per altre informazioni sull'emulazione del mainframe e sui servizi, vedere il Centro migrazione [del mainframe di Azure.](https://azure.microsoft.com/migration/mainframe/)

## <a name="migrate-mainframe-closer-to-azure"></a>Eseguire la migrazione del mainframe più vicino ad Azure

- [Compilatore ASNA Visual RPG per .NET](https://asna.com/us/products/visual-rpg) .NET Framework tramite Visual Studio plug-in.
- [Ambiente di sviluppo Asysco AMT COBOL](https://www.asysco.com/cobol/) (Unisys, mainframe IBM e altri dialetti COBOL come Micro Focus COBOL).
- Architettura di distribuzione basata sul cloud [Asysco AMT GO](https://www.asysco.com/amt-go/) per carichi di lavoro di fascia alta.
- [Asysco AMT Transform](https://www.asysco.com/amt-transform/) per la conversione di dati, codice, script, sicurezza, interfacce e altri artefatti mainframe.
- [Strumenti di sviluppo e integrazione di Fujitsu NetCOBOL.](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/)
- [Strumenti di sviluppo e integrazione di Micro Focus Visual COBOL.](https://www.microfocus.com/products/visual-cobol/)
- [Compilatore legacy PL/I di Micro Focus](https://www.microfocus.com/campaign/download/pli-modernization/) per la piattaforma .NET, che supporta la sintassi PL/I del mainframe, i tipi di dati e il comportamento.
- Piattaforma di integrazione mainframe di [Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- [Strumenti di sviluppo e integrazione di Modern Systems CTU (COBOL-To-Universal).](https://modernsystems.com/automatic-cobol-to-java-conversion/)
- [Strumenti di sviluppo e integrazione di NTT Data Enterprise COBOL.](https://us.nttdata.com/en/digital/application-development-and-modernization)
- [Compilatore legacy NTT Open PL/I](https://us.nttdata.com/en/digital/application-development-and-modernization) per la piattaforma .NET, che supporta la sintassi PL/I del mainframe, i tipi di dati e il comportamento.
- [Strumenti di sviluppo e integrazione del compilatore Raincode COBOL.](https://www.raincode.com/products/cobol/)
- [Il compilatore PL/I raincode](https://www.raincode.com/products/pli/) per la piattaforma .NET supporta la sintassi PL/I del mainframe, i tipi di dati e il comportamento.
- [Compilatore ASM370 di Raincode](https://www.raincode.com/technical-landscape/asm370/) per la sintassi assembler 370 e HLASM mainframe.

- Piattaforma di integrazione mainframe di [Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- Ambiente di elaborazione delle transazioni nativo dell'ambiente di [sviluppo NTT DATA Mainframe.](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf)
- [NTT DATA Batch Processing Environment](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) (BPE), incluse le funzionalità di transazione JCL.
- [Emulatore CICS di Raincode](https://www.raincode.com/technical-landscape/cics/) per le piattaforme .NET e Azure.
- Interprete JCL compatibile con il plug-in [Raincode.](https://www.raincode.com/products/jcl/)

- Servizi end-to-end [asysco AMT per](https://www.asysco.com/migration-services/) progetti di migrazione, tra cui inventario e analisi, training di progettazione, prove di abbigliamento, supporto go-live e post-migrazione.

- [Heirloom Computing](https://www.heirloomcomputing.com/tag/convert-cobol-to-java/) services per convertire mainframe COBOL, CICS e VSAM in Java.
- Contenitore software gestito [LzLabs Software Defined Mainframe](https://www.lzlabs.com/) per la migrazione di applicazioni mainframe a computer Linux o ambienti cloud privati, pubblici e ibridi.
