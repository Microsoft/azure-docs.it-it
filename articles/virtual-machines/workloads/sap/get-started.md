# <a name="use-azure-to-host-and-run-sap-workload-scenarios"></a>Usare Azure per ospitare ed eseguire scenari di carico di lavoro SAPUse Azure to host and run SAP workload scenarios

Quando si usa Microsoft Azure, è possibile eseguire in modo affidabile i carichi di lavoro e gli scenari SAP mission-critical su una piattaforma scalabile, conforme e collaudata dall'azienda. Ottieni scalabilità, flessibilità e risparmi sui costi di Azure. Con la partnership ampliata tra Microsoft e SAP, è possibile eseguire applicazioni SAP tra gli scenari di sviluppo e test e produzione in Azure ed essere completamente supportati. Da SAP NetWeaver a SAP S/4HANA, SAP BI su Linux a Windows e da SAP HANA a SQL, ti abbiamo coperto.

Oltre a ospitare scenari SAP NetWeaver con dbMS diversi in Azure, è possibile ospitare altri scenari di carico di lavoro SAP, ad esempio SAP BI in Azure.Other hosting SAP NetWeaver scenarios with the different DBMS on Azure, you can host other SAP workload scenarios, like SAP BI on Azure. 

L'unicità di Azure per SAP HANA è un'offerta che distingue Azure.The uniqueness of Azure for SAP HANA is an offer that sets Azure apart. Per abilitare l'hosting di più scenari SAP che richiedono memoria e CPU che coinvolgono SAP HANA, Azure offre l'uso di hardware bare metal dedicato al cliente. Utilizzare questa soluzione per eseguire distribuzioni SAP HANA che richiedono fino a 24 TB (120 TB scale-out) di memoria per S/4HANA o altro carico di lavoro SAP HANA. 

L'hosting di scenari di carico di lavoro SAP in Azure può anche creare requisiti di integrazione delle identità e Single Sign-On.Hosting SAP workload scenarios in Azure also can create requirements of identity integration and single sign-on. Questa situazione può verificarsi quando si usa Azure Active Directory (Azure AD) per connettere diversi componenti SAP e offerte di software SAP come servizio (SaaS) o platform-as-a-service (PaaS). Un elenco di tali scenari di integrazione e Single Sign-On con le entità Azure AD e SAP è descritto e documentato nella sezione "Integrazione e Single Sign-On dell'identità SAP di AAD".
