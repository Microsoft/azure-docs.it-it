# <a name="comparing-appsource-and-the-azure-marketplace"></a>Confronto tra AppSource e Azure Marketplace
Le vetrine di [Azure Marketplace](https://azuremarketplace.microsoft.com) e [Microsoft AppSource](https://appsource.microsoft.com) soddisfano esigenze specifiche dei clienti e consentono di rivolgersi ai clienti in base al ruolo. Permettono infatti di offrire la soluzione o il servizio più adatto in base al tipo di cliente.

## <a name="understanding-the-differences-between-storefronts"></a>Differenze tra le vetrine
Quando si sceglie una vetrina è necessario prima di tutto identificare i destinatari della propria offerta. Se ci si rivolge ai destinatari di più vetrine, è possibile pubblicare una sola volta per la vendita in tutte le vetrine.

| Azure Marketplace | AppSource |
|:--- |:--- |
| Sviluppatori e professionisti IT | Utenti aziendali e decision maker aziendali |
 
Le tabelle seguenti descrivono i vantaggi che è possibile conseguire usando una vetrina:

| Vantaggi | Azure Marketplace | AppSource | 
|:--- |:--- | :--- |
| **Flessibilità di fatturazione** | Azure Marketplace supporta i tipi di offerta gratuita, di valutazione, BYOL e transazionale. Le offerte transactable sono basate sul consumo o su sottoscrizione. Le offerte basate sul consumo vengono fatturate in base all'utilizzo orario e possono fornire un periodo gratuito iniziale. I servizi in abbonamento vengono fatturati mensilmente o annualmente a tariffa fissa o per posto.| AppSource supporta tipi di offerta gratuita, di prova, BYOL e transazionale. Le offerte effettuabili in abbonamento sono prodotti SaaS basati su abbonamento che possono essere fatturati mensilmente o annualmente per postazione o tariffa forfettaria. |
|**Possibilità di collegamento con altri partner**  | Attualmente non è possibile collegare all'offerta un provider di servizi o partner di distribuzione. | Collega fornitori di software indipendenti, integratori di sistemi e provider di servizi gestiti a specifici scenari di implementazione. È possibile vendere in modalità collaborativa a nuovi clienti. |
| **Automazione** | Attualmente non è possibile collegare all'offerta un provider di servizi o partner di distribuzione. È possibile sfruttare il provisioning automatizzato di soluzioni SaaS con componenti aggiuntivi e usare l'opzione di pubblicazione App di Azure: Modello di soluzione per automatizzare gli scenari di distribuzione e raccolta di dati basati su SaaS. | Collega fornitori di software indipendenti, integratori di sistemi e provider di servizi gestiti a specifici scenari di implementazione. È possibile vendere in modalità collaborativa a nuovi clienti. |
| **Più tipi di cloudMultiple cloud types** | Usando i tipi seguenti, è possibile pubblicare soluzioni sia nel cloud pubblico che in locale.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>Cloud regionali, tra cui Cina e in Germania</li></ul> | Non supporta attualmente Azure Stack, Azure per enti pubblici o cloud internazionali. |
| **Presentazione contestuale ai clienti** | È possibile rendere disponibile la propria soluzione all'interno del portale di Azure per ricerche contestuali.
