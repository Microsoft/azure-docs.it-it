# <a name="power-bi-app-offer"></a>Offerta per app Power BI

|              |                                |
|--------------|--------------------------------|
| In questo articolo viene illustrato come pubblicare un'app Power BI nel marketplace di Microsoft [AppSource.](https://appsource.microsoft.com/)  Un'app Power BI crea pacchetti di contenuto Power BI personalizzabile, inclusi set di dati, report e dashboard. È quindi possibile distribuire l'app ad altri tenant di Power BI tramite AppSource, eseguire le regolazioni e le personalizzazioni consentite dallo sviluppatore e connetterla ai propri dati. | ![Icona di Power BI](./media/powerbi-icon.png) |


Questo articolo è diviso in tre parti principali:

-   [Prerequisiti](./cpp-prerequisites.md). Requisiti tecnici e aziendali per la creazione e la pubblicazione di un'offerta power BI App.
-   [Creare un'offerta dell'app Power BI.](./cpp-create-offer.md) Come creare una voce dell'offerta di Power BI App tramite il [portale Cloud Partner](https://cloudpartner.azure.com).
-   [Pubblicare un'offerta dell'app Power BI.](./cpp-publish-offer.md) Come inviare una nuova offerta ad AppSource per la pubblicazione e come aggiornare un'offerta esistente.


## <a name="publishing-steps"></a>Procedura per la pubblicazione

Ecco i passaggi generali per la pubblicazione di un'offerta di Power BI App:Here are the high-level steps for publishing a Power BI App offer:

![I passaggi per la pubblicazione dell'app Power BI offrono](media/publishing-steps.png)

Ecco il processo di pubblicazione dell'offerta di Power BI:Here's the Power BI App offer publishing process:

1. Creare un'applicazione modello in Power BI. Questa azione genera un URL di installazione del pacchetto, che rappresenta l'asset tecnico principale per l'offerta. Anche in questo momento, promuovere il pacchetto di test in pre-produzione. Per altre informazioni, vedere Che cosa sono le app modello di [Power BI?](https://docs.microsoft.com/power-bi/service-template-apps-overview). 
2. Raccogli o crea i materiali di marketing dell'offerta, tra cui: nome ufficiale, descrizione, loghi, ecc. 
3. Raccogliere o creare i documenti legali e di supporto dell'offerta: *condizioni d'uso,* *informativa sulla privacy,* politica di *supporto,* assistenza per gli utenti, ecc.
4. Creare l'offerta: utilizzare il portale Cloud Partner per configurare i dettagli dell'offerta, tra cui la descrizione dell'offerta, i materiali di marketing, le informazioni legali, le informazioni di supporto e le specifiche delle risorse.  Dopo aver specificato l'offerta, inviarla per la pubblicazione.
5. Monitorare il processo di pubblicazione nel portale Cloud Partner.  Durante questo passaggio, il team di onboarding di AppSource testa, convalida e certifica l'applicazione. 
6. Dopo che l'app è stata certificata con successo, esaminarla nel proprio ambiente di test e rilasciarla. 
7. L'app Power BI è elencata in AppSource (va in diretta").
8. In Power BI alzare di livello il pacchetto di pre-produzione nell'ambiente di produzione. Per altre informazioni, vedere [Gestire la versione dell'app modello.](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release)
