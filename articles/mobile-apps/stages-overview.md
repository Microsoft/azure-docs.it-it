# <a name="different-stages-in-mobile-application-development"></a>Diverse fasi nello sviluppo di applicazioni mobili
La creazione di un'applicazione mobile prevede più fasi. È possibile sfruttare facilmente i servizi, gli strumenti e le tecnologie Microsoft per creare applicazioni per dispositivi mobili di alto livello.

## <a name="app-development"></a>Sviluppo di app
È possibile creare applicazioni client native a piattaforma singola con linguaggi come Objective-C e Java.You can build native single-platform client applications with languages like Objective-C and Java. Puoi creare app multipiattaforma usando Xamarin, React Native e Unity. È anche possibile creare app ibride usando Cordova.You can also build hybrid apps by using Cordova. Per costruire applicazioni client, utilizzare gli IDE e gli editor di codice, ad esempio Visual Studio, Visual Studio per Mac o Visual Studio Code in PC e Mac.

## <a name="plan-and-design-with-back-end-services"></a>Pianificare e progettare con i servizi back-end
 Microsoft offre un'ampia gamma di servizi per la configurazione di un back-end sicuro, scalabile e affidabile per l'applicazione mobile. È possibile creare API e trigger senza server senza dover gestire i propri server. Usa le notifiche push, la sincronizzazione dei dati offline, l'autenticazione e i servizi dati per formare un back-end mobile completo ed esteso che supporta una varietà di scenari di costruzione di app. 
 
 La creazione di un servizio back-end ha molti requisiti. È necessario:
   - Limitare l'archiviazione dei dati sul dispositivo.
   - Sincronizza i dati su più dispositivi.
   - Assicurarsi che l'applicazione venga eseguita senza connettività di rete.
   - Inviare notifiche agli utenti.
   - Ridurre il consumo della batteria.
   - Crea un back-end scalabile, affidabile e sicuro.
   - Automatizzare la manutenzione del server.

## <a name="devops-and-continuous-monitoring"></a>DevOps e monitoraggio continuo
Per creare applicazioni mobili e incentrate sul cloud, sfruttare l'integrazione continua (CI), la distribuzione continua (CD) e i servizi di monitoraggio continuo per produrre compilazioni native per tutte le piattaforme. Puoi eseguire test di automazione dell'interfaccia utente su migliaia di dispositivi fisici, automatizzare le versioni tramite canali di beta testing o distribuirli direttamente negli app store. È inoltre possibile raccogliere informazioni sull'analisi dei dati di utilizzo, sugli errori e sulle eccezioni per le applicazioni.

## <a name="additional-services"></a>Servizi aggiuntivi
Molti servizi di Azure possono essere utilizzati a seconda del tipo e della natura dell'applicazione in fase di compilazione:Many Azure services can be consumed depending on the type and nature of application being built:
  - Ricerca cognitiva di Azure per eseguire ricerche in tutti i tipi di dati interni, con query rapide.
  - Servizi cognitivi di Azure per usare l'intelligenza artificiale nell'applicazione.
  - Azure SignalR per abilitare la comunicazione in tempo reale nell'applicazione.
