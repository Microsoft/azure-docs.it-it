Azure Data Lake Analytics semplifica la complessa attività di gestione dell'infrastruttura distribuita e del codice complesso. Esegue il provisioning dinamico delle risorse ed è possibile utilizzarle per eseguire analisi su exabyte di dati. Al termine del lavoro, le risorse vengono escone automaticamente. Si paga solo per la potenza di elaborazione utilizzata. Quando si aumentano o diminuiscono le dimensioni dei dati archiviati o la quantità di calcolo utilizzata, non è necessario riscrivere il codice. Per aumentare i limiti predefiniti per la sottoscrizione, contattare il supporto tecnico.

| **Risorsa** | **Limite** | **Commenti** |
| --- | --- | --- |
| Numero massimo di processi simultanei |20 | |
| Numero massimo di unità di analisi (AUs) per account |250 | Usare qualsiasi combinazione fino a un massimo di 250 unità di analisi in 20 processi. Per aumentare questo limite, contattare il supporto tecnico Microsoft.To increase this limit, contact Microsoft Support. |
| Dimensioni massime degli script per l'invio di processi | 3 MB | |
| Numero massimo di account data Lake Analytics per area geografica | 5 | Per aumentare questo limite, contattare il supporto tecnico Microsoft.To increase this limit, contact Microsoft Support. |
